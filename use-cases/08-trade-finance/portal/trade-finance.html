Cross-Region Design

🏦 me-south-1 (Bahrain): Data residency — banking data stays in-region
🧠 eu-west-1 (Ireland): AI models — Claude Sonnet 4 available here

Data never leaves Bahrain — only SQL results (JSON) cross the region boundary. VARBINARY blobs are decoded in me-south-1 and only text previews are returned.

How SQL Generation Works

100% dynamic — zero hardcoded queries. The agent:
1. Discovers schema → calls get_schema_info
2. Writes SQL → Claude generates a SELECT query
3. Executes → calls execute_sql_query
4. Interprets → Claude reads results and formulates answer
